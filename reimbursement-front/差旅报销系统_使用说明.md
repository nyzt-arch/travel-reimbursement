# 差旅费用报销系统 — 使用说明

> 本文档基于概要设计文档及页面原型，详细说明系统的每一个页面布局和操作方法，面向开发人员和测试人员阅读。

---

## 一、系统概述

### 1.1 适用场景

本系统用于员工因公出差后向企业申请报销差旅补助，主要涵盖：

- 员工自行垫付差旅费用后的报账结算
- 餐费、交通、通讯等差旅补助的逐日核算
- 费用按比例分摊到不同归属公司/项目

### 1.2 单据状态说明

| 状态 | Code | 触发条件 | 说明 |
|------|------|----------|------|
| **草稿** | 0 | 新增报销单后 | 数据未填完，可继续编辑 |
| **已完成** | 1 | 点击提交并通过校验 | 报销单填写完成，等待处理 |
| **已作废** | 2 | 手动执行作废操作 | 单据失效，不可恢复 |

### 2.2 查询条件区（页面顶部）

查询条件分两行排列，支持多字段组合搜索。

**第一行（4个条件）：**

| 字段 | 输入方式 | 说明 |
|------|----------|------|
| **报销单号** | 手动输入框 | 输入完整或部分报销单号（如 `RCBX20260515002`） |
| **标题** | 手动输入框 | 按报销标题关键字模糊搜索 |
| **事由** | 手动输入框 | 按出差事由关键字模糊搜索 |
| **费用归属公司** | 下拉选择 | 从下拉菜单中选择公司（北京/上海/武汉/杭州/荆州分公司） |

**第二行（3个条件 + 3个操作按钮）：**

| 字段/按钮 | 类型 | 说明 |
|-----------|------|------|
| **报销部门** | 下拉选择 | 选择报销所属部门（客户成功/企业消费/航旅等事业部） |
| **报销人** | 下拉选择 | 选择具体的报销员工 |
| **业务类型** | 树形下拉选择 | 按三级树结构选择业务类型（如：员工差旅活动 → 境内出差 → 项目出差） |
| **新增** 按钮 | 蓝色主操作按钮 | 点击跳转到报销单详情页，创建新的报销单 |
| **清除** 按钮 | 普通按钮 | 一键清空所有查询条件输入框 |
| **搜索** 按钮 | 蓝色主操作按钮 | 根据当前填写的查询条件执行搜索，刷新下方列表 |

### 2.3 列表数据表格

表格按创建时间倒序排列，每行展示一条报销单记录。

| 列名 | 展示要求 | 说明 |
|------|----------|------|
| **序号** | 纯数字 | 当前页内的行号（1、2、3...） |
| **操作** | 图标按钮 | 见下方 §2.4 行内操作说明 |
| **报销单号** | 蓝色链接 | 点击跳转至报销单详情页；格式：`RCBX` + 年月日 + 序号 |
| **单据状态** | 文字标签 | 草稿（蓝色）/ 已完成（绿色）/ 已作废（灰色）/ 审批中（橙色） |

| **报销部门** | 文字 | 展示格式：`[部门编号]部门名称`，如 `[CS001]测试部` |
| **费用归属公司** | 文字 | 公司名称，如 `初始化测试` |
| **业务类型** | 文字 | 叶子节点业务类型名称，如 `日常办公`、`项目出差` |
| **报销标题** | 蓝色链接 | 点击跳转至报销单详情页 |
| **报销事由** | 文字（超出省略） | 超长内容截断显示，鼠标悬停可查看完整内容 |
| **补助金额** | 数字，右对齐 | 展示该报销单的补助总金额，单位默认为 CNY |

## 五、分区二：补录行程

### 5.1 分区头部

- 标题：**「补录行程」**
- 右侧：**⊕ 补录行程** 蓝色文字按钮 + **∧** 折叠箭头
- 点击「补录行程」按钮，弹出行程录入弹框

### 5.2 行程列表

列表字段说明：

| 列名 | 说明 |
|------|------|
| **序号** | 行内序号 |
| **出行人员** | 格式：`姓名/工号`，如 `徐年年/202101497` |
| **出差日期** | 格式：`YYYY-MM-DD 至 YYYY-MM-DD` |
| **行程** | 格式：`出发城市 - 到达城市`，如 `武汉 - 北京` |
| **行程说明** | 用户填写的行程文字说明 |
| **操作** | 🗑️ 删除 / ✏️ 编辑 / 📋 复制，三个图标按钮 |

### 5.3 行程弹框（新增/编辑/复制）

![补录行程弹框](C:/Users/admin/.gemini/antigravity-ide/brain/8b01cdad-e63a-4d98-90c7-80b31c8939fb/image_20.png)

弹框标题：**「补录行程」**，右上角有 ✕ 关闭按钮。

**顶部提示信息（橙色警告条）：**

> 仅可补录未从申请单带入或未产生费用的行程信息  
> 跨天跨城行程填写说明：出发城市-到达城市：武汉-北京；出发日期-到达日期：1号-5号；1号~5号补助按北京匹配

**表单字段：**

| 字段 | 输入方式 | 规则 |
|------|----------|------|
| **出行人** * | 下拉选择 | 必填，从员工列表选择 |
| **出发城市** * | 下拉选择（带清除按钮） | 必填，从城市列表选择 |
| **到达城市** * | 下拉选择（带清除按钮） | 必填，从城市列表选择 |
| **出发到达日期** * | 日期范围选择器 | 必填；到达日期 ≥ 出发日期，且不可晚于当前日期 |
| **行程说明** * | 多行文本输入框 | 必填，最多 500 字 |

**底部按钮：**
- **取消**：关闭弹框，不保存
- **保存**：校验必填 + 日期不重复后保存，并自动在「补助信息」分区中生成对应条目

### 5.4 删除行程确认弹框

![删除行程确认弹框](C:/Users/admin/.gemini/antigravity-ide/brain/8b01cdad-e63a-4d98-90c7-80b31c8939fb/image_19.png)

弹框标题：**「提示」**，内容：「确认删除？」

- **取消**：关闭弹框，不执行删除

> **注意**：复制行程时，弹框会预填当前行数据，保存后生成全新一条（需注意日期重复校验）

- **合计行**：显示当前所有行的比例合计（橙色）和金额合计（橙色 CNY 格式）

分摊比例列表头旁边有 **「均摊」** 蓝色按钮，点击后按当前行数均分比例和金额，余数差值放在第 1 行。

### 8.3 分摊联动规则

| 规则 | 说明 |
|------|------|
| **第 1 行不可手动编辑** | 第 1 行的分摊比例和分摊金额通过联动自动计算，永远不可直接编辑 |
| **第 1 行比例 = 100% - ∑（第2+行比例）** | 修改任何第 2+ 行的分摊比例，第 1 行自动更新 |
| **∑（第2+行）> 100%** | 超出后，当前输入框自动清空该次输入值 |
| **仅有第 1 行时** | 第 1 行默认展示 100%，不可编辑 |
| **均摊除不尽时** | 余数（差值）放在第 1 行 |
| **存储区间** | 分摊比例存储为 0~1（如 0.35），页面展示转为百分比（如 35.00%） |

### 8.4 删除分摊行

**只有 1 行时**：弹提示「至少保留一条分摊信息」，不执行删除。

**有多行时**：弹确认弹框（同行程删除弹框样式），点「确定」删除当前行。

## 十、底部按钮栏（固定吸底）

底部按钮栏**固定在页面最底部**，不随页面滚动。

| 按钮 | 样式 | 操作说明 |
|------|------|----------|
| **关闭** | 普通白色按钮 | 弹出确认弹框（"确认关闭？"），点击「确认」后关闭当前页面，返回列表页 |
| **提交** | 蓝色主操作按钮 | 执行全量数据校验，通过后提示「提交成功」弹框，点击「确认」关闭页面 |

### 提交时触发的校验规则

| 校验项 | 校验内容 |
|--------|----------|
| ① **必填校验** | 报销标题、出差事由、报销人、报销部门、费用归属公司、业务类型均不可为空 |
| ② **补录行程必填** | 每条行程中：出行人、出发城市、到达城市、出发到达日期、行程说明均不可为空 |
| ③ **行程日期不重复** | 同一出行人的多条行程，日期范围不可有任何重叠 |
| ④ **分摊比例合计 = 100%** | 所有分摊行的比例之和必须等于 100% |
| ⑤ **分摊金额合计 = 补助总额** | 所有分摊行的金额之和必须等于费用合计中的补助总金额 |

> [!IMPORTANT]
> 以上校验需要**前端页面**和**后端接口**同时执行，防止绕过前端直接提交。
