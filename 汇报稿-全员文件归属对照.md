# 全员文件归属对照表：老师点到哪个文件/哪段代码 → 谁回答

> **用法**：老师问"这个文件是谁写的？"或者"这段代码你能解释一下吗？"，立刻查表确定谁站起来回答。
> 
> **分工特点**：三人全栈参与，每人都有后端和前端文件。

## 四、快速记忆口诀

```
A 管核心 —— 后端业务全流程（Controller/DTO/VO/Entity/Mapper/ServiceImpl 540行）
            + 前端最复杂页面（详情页 520行 / 分摊 550行 / 数据表格）

B 管基础 —— 后端基础数据平台（BaseData 全套 + 树形算法）
            + 前端架构与组件库（脚手架/Store/Router/API/8个通用组件/CSS体系）

C 管串联 —— 后端公共模块（Result/异常处理/MyBatisPlus配置）+ 4个子表Service
            + 前端业务与工具（列表页/7个业务区块/弹窗/校验/补助计算/格式化）
```

**一句话版：**
- 老师打开 **核心业务 Java 文件**（FkReimMainServiceImpl / ReimbursementController）→ **A** 站起来
- 老师打开 **基础数据 Java 文件**（BaseData*/Base*Entity）→ **B** 站起来
- 老师打开 **公共模块 Java 文件**（Result / GlobalExceptionHandler / MyBatisPlusConfig / 子表Service）→ **C** 站起来
- 老师打开 **详情页 / 分摊 / 数据表格** → **A** 站起来
- 老师打开 **通用组件 / Store / API / 样式 / 配置** → **B** 站起来
- 老师打开 **列表页 / 业务弹窗 / 工具函数** → **C** 站起来
- 老师问 **"你写过后端/前端吗？"** → **三个人都站起来** 😄
